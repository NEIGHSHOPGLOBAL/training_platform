PROJECT OVERVIEW
Training Management & Referral Platform
Frontend: React.js
Backend: Python Flask
==========================================================

PROJECT PURPOSE
----------------------------------------------------------

This platform is built for managing software development training programs along with a complete referral & affiliate earning system.

Students can register, verify their email using OTP, purchase training manually through QR payment, upload payment proof, wait for admin approval, and then access course resources.

Apart from students, the platform also includes an independent Referral Partner Portal where affiliates can earn commission without becoming students.

The system should be clean, modern, secure, scalable and production-ready.

==========================================================
LANDING PAGE REFERENCE
==========================================================

An existing landing page (index.html) is already available and should be used as the primary design reference for the entire platform.

The objective is to maintain a consistent brand identity across the complete application.

The developer should reuse and adapt the following from the landing page wherever applicable:

• Color palette
• Typography
• Buttons
• Icons
• Cards
• Border radius
• Shadows
• Spacing
• Animations
• Backgrounds
• Gradients
• UI Components
• Overall visual language

The React application (Student Dashboard, Admin Panel, Referral Portal, Authentication Pages, etc.) should feel like a natural extension of the landing page rather than a separate product.

The design system should remain consistent throughout the project.

The existing landing page should NOT be rewritten from scratch. Instead, it should be used as the visual foundation while converting the application into a modern React-based dashboard experience.

The landing page should continue to function as the public-facing website, while React powers the authenticated sections of the platform.

The developer should extract reusable design patterns from index.html and create reusable React components to ensure consistency across all pages.

Goal:
A user should immediately recognize that the landing page and dashboard belong to the same product, with identical branding, theme, and overall user experience.

==========================================================
UI/UX EXPECTATION
==========================================================

This project is expected to have a premium, modern SaaS-style user interface rather than a basic CRUD dashboard.

Preferred design characteristics:

• Clean and minimal layouts
• Modern glassmorphism where appropriate
• Smooth animations and transitions
• Responsive on desktop, tablet and mobile
• Beautiful dashboard cards
• Attractive charts and statistics
• Professional tables with search, filters and pagination
• Consistent spacing and typography
• Elegant loading states and skeleton screens
• Toast notifications for all actions
• Confirmation dialogs for destructive actions
• Empty state illustrations/messages
• High-quality user experience with attention to detail

The objective is to deliver a polished production-grade application, not just a functional management system.
