2. STUDENT AUTHENTICATION PORTAL
============================================================

2.1 Student Register / Login Page
- URL: /portal or /student
- Purpose:
  Student entry point using email OTP only.
- Fields:
  - Full name
  - Email
  - Phone number
  - Referral code, optional
- Actions:
  - Send OTP
  - Verify OTP
  - Continue to student dashboard
- Rules:
  - No password login for students.
  - If referral code exists in URL, auto-fill it.
  - Invalid referral code should block registration.
  - Empty referral code is allowed.

2.2 OTP Verification Screen
- Purpose:
  Verify student email before account/session creation.
- Fields:
  - OTP code
- Actions:
  - Verify OTP
  - Resend OTP
- Development behavior:
  - OTP may be shown on screen in development.
- Production behavior:
  - OTP should be sent by email through SMTP.

3. STUDENT PORTAL
============================================================

3.1 Locked Student Dashboard
- Purpose:
  First dashboard state after student login before payment approval.
- UI behavior:
  - Dashboard content remains blurred/locked.
  - Center card shows "Enroll to Get Access".
- Student can:
  - View enrollment pricing
  - Choose online/offline mode
  - View QR/payment instructions
  - Upload payment screenshot
- Status shown:
  - Registered
  - Payment Submitted
  - Under Review
  - Rejected
- Notes:
  If rejected, student can upload payment proof again.

3.2 Payment Submission Screen
- Purpose:
  Allow student to submit training payment proof.
- Fields:
  - Training mode: Online or Offline
  - Payment screenshot upload
- Pricing display:
  - Online original price
  - Online referral discounted price
  - Offline original price
  - Offline referral discounted price
- Actions:
  - Upload payment screenshot
  - Submit for review
- Rules:
  - Payment approval is done outside student portal by admin.
  - Student dashboard unlocks only after approval.
